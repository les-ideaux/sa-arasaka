ADR-01 \
Titre : Choix au format Application Web \
Contexte : Nous devons choisir le format de l'application (Mobile, Web, Autre...) \
Décision : Dans le contexte du développement de l'application et pour un MVP rapide, utilisable sur PC et Mobile, une application web s'avère pertinente; cela permet aussi de tester rapidement les fonctionnement.

ADR-02 \
Titre : Choix d'Angular pour la stack Front-End \
Contexte : Nous devons choisir un framework Front-End \
Décision : Nous avons choisis Angular comme framework front-end étant donné que nous sommes habitué à ce framework et qu'il offre des outils pour un développement rapide d'application avec une listes d'outils déjà intégré.

ADR-03 \
Titre : Choix de SpringBoot Java pour la stack Back-End \
Contexte : Nous devons choisir un framework Back-End \
Décision : Nous avons choisis SpringBoot, car il permet de se concentrer sur la logique métier car il fournit une couche d'abstraction importante pour l'implémentation d'API. Notre groupe est également familier avec cette stack.

ADR-04 \
Titre : Choix de sqlite3 pour la base de données \
Contexte : Nous devons choisir un format de BDD \
Décision : Nous avons choisis SQLite pour sa legereté et ajoute une facilité au niveau du développement. Dans le contexte du développement (MVP) cela correspond très bien à nos besoins.
